# M1 PWA · Working Code · Ноев Ковчег

## Что это

Полностью рабочее Progressive Web Application модуля **M1 «Витрина и калькулятор»** Платформы «Ноев Ковчег». Открывается в любом современном браузере на iPhone, Android, iPad, ноутбуке. Устанавливается на главный экран как нативное приложение.

## Как открыть

### Локально на компьютере
1. Откройте `index.html` двойным кликом — приложение запустится в вашем браузере.
2. Калькулятор и форма заявки работают сразу.
3. Переключение языка (RU/EN/HY) и темы (light/dark/auto) работает мгновенно. Выбор сохраняется между сессиями.

### На телефоне / планшете
**Способ 1 (быстрый — через локальный сервер):**
```
# Из этой папки запустите:
python -m http.server 8080
# или
npx serve -p 8080
```
Затем откройте на телефоне в браузере: `http://[ваш-локальный-IP]:8080`

**Способ 2 (для production):** разместите содержимое папки на любом статическом хостинге (бесплатные варианты: Netlify Drop, Cloudflare Pages, GitHub Pages, Vercel).

### Установка как нативное приложение

**На iPhone (Safari):**
1. Откройте приложение в Safari
2. Нажмите кнопку «Поделиться» (внизу)
3. Выберите «На экран Домой»

**На Android (Chrome):**
1. Откройте приложение в Chrome
2. Меню (три точки) → «Установить приложение» или «Добавить на главный экран»

**На Windows / macOS (Edge / Chrome):**
1. В адресной строке появится иконка установки — нажмите её
2. Или меню → «Установить Ноев Ковчег»

## Деплоймент в production

Размещение PWA на бесплатных хостингах:

### Netlify Drop (самый простой)
1. Откройте https://app.netlify.com/drop
2. Перетащите эту папку в окно браузера
3. Через 30 сек получите URL вида `https://something.netlify.app`

### Cloudflare Pages
```
npm install -g wrangler
wrangler pages deploy . --project-name=noah-ark-pwa
```

### GitHub Pages
1. Создайте public-репозиторий, загрузите эту папку
2. Settings → Pages → Source: main / root → Save
3. URL: `https://[username].github.io/[repo]/`

### Vercel
```
npm install -g vercel
vercel --prod
```

Все варианты — **бесплатно**, лимиты с запасом для проекта Phase 1.

## Что приложение НЕ делает (по дизайну M1)

- ❌ KYC-проверку — это M2
- ❌ Реальный приём платежей — это M3
- ❌ Кошелёк / orderbook — это M3
- ❌ Регуляторный портал — это M4

M1 — **витрина и лид-капчер**. Обращения с формы заявки сохраняются локально на устройстве (для production: подключить любой backend форм — Formspree, Web3Forms, ваш API).
