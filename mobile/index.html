Мобильное приложение «Ноев Ковчег»

Кросс-платформенное приложение iOS / Android для трёх типов пользователей: собственников залоговой недвижимости (РА + диаспора), инвесторов и регулятора-наблюдателя (ЦБ РА). Фазированная архитектура: Phase 1 — PWA, Phase 2 — React Native с native-модулями.

10 модулей приложения

00
Design spec и продукт-стратегия
UX-концепт (Арарат + золото + пергамент), персоны, journey-карты, KPI Phase 1.
M1
Showcase и калькулятор
Витрина проектов платформы, расчёт доходности для собственника и инвестора, ввод в воронку.
M2
Кабинет собственника
Заведение объекта, KYC, e-подпись Декларации об отказе от прав, статус по фазам, купоны и страховка.
M3
Кабинет инвестора
Senior- и Junior-токены, портфель, выплаты, KYC / AML, налоговая документация по юрисдикции инвестора.
M4
Регулятор-портал
Read-only доступ ЦБ РА к ключевым метрикам пула, NAV, риск-аналитике, аудит-трейлу.
05
API-спецификация
OpenAPI 3.1, контракты между приложением и платформой, схемы данных, оркестрация custody / KYC / NAV.
06
Working PWA
Запускаемая Progressive Web App: HTML/CSS/JS + manifest + service-worker, оффлайн-кэш ключевых экранов.
07
React Native source
App.tsx + src/, Expo / EAS-сборка, готова к публикации в App Store и Google Play (Phase 2).
08
Marketing / store assets
Иконки 1024×1024, скриншоты экранов под все требуемые форматы магазинов, описания.
09
Deployment и submission
Чек-листы EAS-сборки, App Store Connect, Google Play Console, политика конфиденциальности.

Текущий статус

  • Phase 1 (PWA) — рабочее демо доступно по ссылке выше; полный source — в репо проекта.
  • Phase 2 (React Native) — исходный код App.tsx + src/ + EAS-конфиг подготовлен; продакшн-сборка ожидает MVP-cut по бэкенду платформы.
  • Backend — описан в C-2 Технической архитектуре и API-спецификации (модуль 05). MVP-имплементация — следующий блок дорожной карты после G1.
