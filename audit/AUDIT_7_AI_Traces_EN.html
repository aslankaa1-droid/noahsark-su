Noah’s Ark Platform — independent editorial audit on “AI traces”

Effective Date 11 May 2026
Auditor Helena Marsh, Senior Editor (ex-Financial Times, ex-Wall Street Journal)

Methodology Reuters Institute AI provenance framework + FT/WSJ editorial style guide + the client’s (A-Kh. A. Kagirov) 10-category classification (A–J)

1. EXECUTIVE SUMMARY

Overall verdict: 6.5 / 10 of “AI cleanliness”.

The first wave of editing (10–11 May) had a meaningful effect on the valuation report Оценка_Ноев_Ковчег_2026-05-11_RU.md and its three translations. That document is now close to Big4 tone (≈ 9/10). However, outside the perimeter of the valuation report, the remaining 25+ project files are nearly untouched: the version stigmata Version 1.0 survive in the headers of 15 files, ≈ 245 lines with emoji are scattered across 33 .md files, pompous phrases like “the only in the world”, “the first in the world”, “no analogue” survive in the pitch deck, the speech, the mobile strategy, and the one-pager. The client has expressly formulated the requirement “everywhere the user sees” — that requirement has not yet been satisfied.

The three most frequent patterns (in descending order):

  1. Emojis in final documents — 245 lines in 33 .md files. The heaviest — E2_Инвестиционный_pitch_deck.md (38), A_Концепт (26), 00_Product_Strategy_RU.md (12), the mobile app README.md (10), every Module spec (3–4 each).
  2. Версия 1.0 / Version 1.0 in the header — 15 mobile-app files (Product Strategy RU/EN, all four Module Specs RU/EN, API Spec RU/EN, store_listing RU/EN, PWA README, React Native README, and pre-cleanup mentions in drafts).
  3. “Unique” / “the only in the world” / “the first in the world” / “no analogue” — found in 8 key client documents (A, C, D, E one-pager, E2 deck, README, Product Strategy, press kit).

Three priority actions (in detail — § 3):

  1. PRIORITY-1 (Critical, ~30 min of work): Remove the line **…Версия 1.0…** from the header of 15 mobile-app .md files. This is the identical AI marker the client already rejected in chat on 10 May. A sed-friendly rule is in § 3.A.
  2. PRIORITY-2 (Critical, ~45 min): Run E2_Инвестиционный_pitch_deck.md through an editorial pass — all 38 emoji lines (🇦🇲 💰 🌍 🏠 ✅ 📜 🏛 🌐 ⚙️ 📊 1️⃣ 2️⃣ … 7️⃣ 🎯 💎 🤝 📈 📌) — remove or replace with plain unicode bullets (•). Likewise — A_Концепт, D1_Whitepaper, the mobile docs.
  3. PRIORITY-3 (High, ~30 min): Replace “the only in the world”, “the first in the world”, “no analogue” with specific phrasings (“a new combination of four elements”, “no direct analogues found”). In 8 files.

After these three passes, the verdict rises from 6.5 to ≈ 8.5/10. To push to 9.5+, the D_Спичрайт speech and Сделанное_и_Сомнительное.md (pomposity, “the most secured”, “the most progressive”, “a level of transparency unavailable to any classical financial institution”) need editing — that is a fourth pass, another ≈ 40 min.

Regulatory citation texts (HO-159-N HTML, references to the law’s articles, CBA Regulation texts) must not be touched — that is normative material, and any “fix” there is an error.

3. PATTERN REPLACEMENT CATALOG

3.A. Versioning (sed-friendly)

PATTERN: ' · Версия 1.0 · '
REPLACE: ' · '
SCOPE:   all 15 files from § 2.4.1
PATTERN: ' · Version 1.0 · '
REPLACE: ' · '
SCOPE:   EN variants of Module specs / API spec / Product Strategy / store_listing
PATTERN: '^- Версия 1\.0\s*$'
REPLACE: (delete the whole line)
SCOPE:   06_Working_PWA/README.md line 129; 07_React_Native_Source/README.md line 171
PATTERN: 'NK-PWA-001/2026'   (in service-worker.js → CACHE_VERSION)

3.B. Pomposity (manual replace)

'unprecedented'                             → delete or 'significant'
'revolutionary'                             → 'new'
'innovative'                                → 'new' / delete
'breakthrough'                              → delete
'unique offer'                              → 'offer' (plain)
'unique product for 7 million Armenians'    → 'a product designed for 7 million diaspora Armenians'
'the only instrument in the world'          → 'the instrument' (plain)
'the first project in the world combining…' → 'a project combining…'
'the first opportunity in history'          → 'an opportunity'
'without analogue'                          → 'without direct analogues'
'no analogue exists'                        → 'no direct analogue exists as of the document date'
'the most progressive in our region'        → 'one of the newest comprehensive in the region'
'the most secured instrument'               → 'an instrument with multi-layer protection'
'the most secured digital asset'            → 'a digital asset with real collateral, insurance, and sovereign guarantee'
'no analogues to this architecture in the world today' → 'no direct analogues to this architecture have been found as of the document date'

'outstanding'                               → delete
'inimitable'                                → delete

3.C. AI templates (sed-friendly)

PATTERN: 'It is worth noting that X'         → 'X'
PATTERN: 'It is important to understand that X' → 'X'
PATTERN: 'In this section we shall examine…' → drop the intro; start with the fact
PATTERN: 'Not only X, but also Y'             → 'X. Also, Y.' / 'X plus Y.'
PATTERN: 'In conclusion, it should be noted' → delete (start with the direct conclusion)
PATTERN: 'It should be emphasised that'      → delete
PATTERN: 'It is necessary to note that'      → delete
PATTERN: 'It is worth stressing'             → delete

In the “Noah’s Ark” project these patterns have already been removed in the first wave — but if new text is added, these rules should apply automatically.

4.4 When an AI-trace is not a trace but a real artefact

Not every mention of AI tools in the Noah’s Ark project is an AI trace:

  • Legitimate: B_Декларация_об_отказе_от_прав.md references the 2023 USCO “Works Containing Material Generated by Artificial Intelligence” as a citation of a normative act. This is legally correct and necessary.
  • Legitimate: E2_Инвестиционный_pitch_deck.md slide 11 “Intellectual property” mentions the use of automation tools — this is part of the IP strategy.
  • Legitimate: 06_Working_PWA/i18n/translations.js has a UI label settings_version: 'Version' — this is a mobile-app label, not an AI trace.

The principled distinction: an AI trace = an implicit fingerprint of the tool, revealing its participation. A direct mention of AI tools as the subject (IP, copyright) or their use in a designated UI context (the “Version” label in settings) is not a trace.
